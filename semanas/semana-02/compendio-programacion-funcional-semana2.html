¿Por qué estudiar funciones?

De la matemática al código

Las funciones matemáticas establecen una relación determinista entre entrada y salida. La programación funcional replica esta idea para lograr sistemas previsibles.

Modulización

Separar lógica en funciones puras facilita pruebas, refactorización y composición. El código se vuelve una colección de transformaciones.

Trabajo colaborativo

Funciones sin efectos secundarios evitan sorpresas en equipo y preparan la base para flujos concurrentes seguros.

Funciones matemáticas vs. funciones en código

“Una función es capaz de generar efectos —escribir un archivo, dibujar una imagen— algo inaudito en el mundo matemático.”

Reddit, r/learnmath
Función (estilo matemático) Función (en programación imperativa)
Depende exclusivamente de sus variables de entrada. Puede depender de estado global o mutar parámetros.
Dominios y codominios definidos e inmutables. Limitada por la arquitectura (precisión, memoria).
No produce efectos secundarios. Puede generar efectos: E/S, mutaciones, llamadas a red.

Funciones puras

Determinismo

Mismos argumentos, mismo resultado. Facilita reproducibilidad y pruebas unitarias.

Sin efectos secundarios

No modifica bases de datos, archivos ni variables externas. Trabaja con datos inmutables.

Combinabilidad

Componer funciones puras produce pipelines declarativos y fáciles de razonar.

// Función pura
const descuento = (precio, porcentaje) => precio * (1 - porcentaje);

// Función impura
let total = 0;
const agregarAlCarrito = (precio) => {
  total += precio; // muta un estado externo
  return total;
};

Funciones como ciudadanas de primera clase

Propiedades esenciales

  • Pueden almacenarse en variables y estructuras de datos.
  • Se envían como argumentos o se retornan desde otras funciones.
  • Convierten a las funciones de orden superior en patrones naturales.

Conceptos clave para avanzar

Orden superior

map, filter, reduce abstraen patrones de iteración y delegan decisiones en lambdas.

Currificación

Transforma funciones de múltiples parámetros en una cascada de funciones de un solo parámetro.

Recursión

Reemplaza bucles mutables con llamadas sucesivas hasta alcanzar un caso base.

Refactorización: sumar pares

Imperativo

function sumarParesImperativo(lista) {
  let total = 0;
  for (let i = 0; i < lista.length; i++) {
    if (lista[i] % 2 === 0) {
      total += lista[i];
    }
  }
  return total;
}

Declarativo / funcional

const sumarParesFuncional = (lista) =>
  lista
    .filter((n) => n % 2 === 0)
    .reduce((acc, n) => acc + n, 0);

Patrones repetibles se encapsulan en funciones de orden superior.

Recursión controlada

La recursión reemplaza bucles mutables y se basa en un caso base que corta la ejecución. Útil para estructuras como árboles o listas inmutables.

// Fibonacci recursivo (didáctico)
const fib = (n) => (n < 2 ? 1 : fib(n - 1) + fib(n - 2));

// Con memoización para entornos reales
const fibMemo = ((cache = {}) => (n) => {
  if (n in cache) return cache[n];
  cache[n] = n < 2 ? 1 : fibMemo(n - 1) + fibMemo(n - 2);
  return cache[n];
})();

Controla el crecimiento de la pila con casos base claros y, cuando sea necesario, optimiza con memoización o versiones de cola.

Expresiones lambda

Las lambdas encapsulan transformaciones puntuales. Son esenciales en pipelines funcionales.

Transformaciones rápidas

const potenciasDeDos = Array.from({ length: 5 }, (_, x) => 2 ** x);
console.log(potenciasDeDos); // [1, 2, 4, 8, 16]

Composición

const normalizar = (texto) => texto.trim().toLowerCase();
const capitalizar = (texto) => texto[0].toUpperCase() + texto.slice(1);

const limpiarNombre = (nombre) => capitalizar(normalizar(nombre));
console.log(limpiarNombre("  lambda lovers "));

Ejemplos prácticos

Pipelines declarativos

Modela transformaciones puras encadenando filter, map y reduce.

const ventas = [
  { vendedor: "Ana", total: 1200, estado: "cerrado" },
  { vendedor: "Luis", total: 400, estado: "abierto" },
  { vendedor: "Ana", total: 800, estado: "cerrado" },
  { vendedor: "María", total: 600, estado: "cerrado" }
];

const totalCerradoPorVendedor = (nombre) =>
  ventas
    .filter((venta) => venta.estado === "cerrado" && venta.vendedor === nombre)
    .map((venta) => venta.total)
    .reduce((acc, total) => acc + total, 0);

console.log(totalCerradoPorVendedor("Ana")); // 2000

La estructura facilita agregar reglas, por ejemplo, transformar totales con impuestos o descuentos sin mutar datos.

Funciones vs. métodos en práctica

Encapsula lógica en funciones puras y deja los métodos como envoltorios que delegan responsabilidades.

const calcularPuntaje = ({ aciertos, errores }) => aciertos * 3 - errores;

const estudiante = {
  nombre: "Sofía",
  aciertos: 18,
  errores: 2,
  puntaje() {
    return calcularPuntaje(this);
  }
};

console.log(calcularPuntaje({ aciertos: 20, errores: 1 })); // 59
console.log(estudiante.puntaje()); // 52

Distinguir la función pura de la interfaz orientada a objetos facilita pruebas aisladas y reutilización en otros contextos.

Currificación y composición

Especializa funciones aplicando argumentos parciales y combínalas para generar pipelines expresivos.

const aplicar = (fn) => (valor) => fn(valor);
const multiplicar = (factor) => (valor) => valor * factor;
const sumar = (incremento) => (valor) => valor + incremento;

const duplicar = multiplicar(2);
const incrementarCinco = sumar(5);

const transformar = (valor) => aplicar(incrementarCinco)(duplicar(valor));

console.log(transformar(10)); // 25

Este patrón prepara la base para librerías de composición más avanzadas (p. ej. Ramda, lodash/fp) y refuerza la idea de funciones como piezas intercambiables.

Checklist de la semana

  • Comprende qué hace pura a una función y por qué importa.
  • Practica map/filter/reduce con lambdas para modelar pipelines.
  • Explora recursión y currificación como herramientas declarativas.
  • Reflexiona sobre cuándo son inevitables los efectos secundarios (I/O) y cómo aislarlos.
