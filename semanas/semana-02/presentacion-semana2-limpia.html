Funciones en programación

Una función es una porción de código separada capaz de causar algún efecto o evaluar un valor.

  • Incorporadas en el lenguaje (como console.log()).
  • Provienen de módulos o bibliotecas.
  • Escritas directamente por el programador.
Sintaxis básica: palabra reservada def (Python) o function (JS), nombre, paréntesis con argumentos, y return para devolver valores.

Matemáticas vs programación

Aunque las funciones de programación tienen parentesco con conceptos matemáticos, existen diferencias fundamentales:

Diferencias clave:
  • Efectos: Las matemáticas solo evalúan; la programación puede causar efectos (archivos, pantalla, sonido).
  • Dominio: Matemáticas maneja infinitos; programación está limitada por arquitectura.
  • Estado: Funciones matemáticas no cambian nada; programación puede modificar variables.
  • Efectos secundarios: Inexistentes en matemáticas; comunes en programación imperativa.

Hacia la programación funcional

La programación funcional busca cerrar esta brecha imitando más de cerca el comportamiento de las funciones matemáticas.

Analogía: como usar una calculadora científica donde cada operación es predecible y no afecta otras operaciones.
Objetivo clave: evitar efectos secundarios y hacer que las funciones sean más predecibles, como en matemáticas.

Programación funcional

  • Paradigma declarativo: se enfoca en describir qué queremos lograr, no cómo.
  • Origen histórico: cálculo lambda (λ) de Alonzo Church en los años 30, antes de las computadoras.
  • Definición: construir programas usando expresiones y funciones puras aplicadas en secuencia.

Lenguajes funcionales

Puramente funcionales
  • Haskell: referencia académica, lazy evaluation.
  • LISP: pionero histórico, sintaxis de paréntesis.
  • Scheme: minimalista, educativo.
Multiparadigma
  • JavaScript: funcional + imperativo, web.
  • Python: funcional + OOP, data science.
  • Scala: JVM + funcional + OOP.
  • F#: .NET funcional, Microsoft.

Funciones puras

Una función pura es aquella que, para un mismo conjunto de parámetros de entrada, siempre produce el mismo resultado.

  • Determinismo: mismo input, mismo output.
  • Sin efectos secundarios: no altera el estado del sistema.
  • Inmutabilidad: depende de datos que no cambian.
Clave: la ausencia de efectos colaterales es lo que define la pureza.

Ejemplo: función pura

Función que siempre produce el mismo resultado con los mismos parámetros:

Analogía: una máquina expendedora que siempre da el mismo producto al presionar el mismo botón.
Ejemplo JS: 

const aplicarDescuento = (precio, porcentaje) => precio * (1 - porcentaje);

// Siempre produce el mismo resultado
console.log(aplicarDescuento(100, 0.2)); // 80
console.log(aplicarDescuento(100, 0.2)); // 80

Ejemplo: función impura

Función que produce efectos secundarios o depende de variables externas:

Analogía: una máquina que da diferentes productos según su estado interno o el momento del día.
Ejemplo JS: 

let totalCarrito = 0;
const agregarAlCarrito = (precio) => {
    totalCarrito += precio; // ¡Efecto secundario!
    return totalCarrito;
};

console.log(agregarAlCarrito(25)); // 25
console.log(agregarAlCarrito(25)); // 50 (diferente resultado)

Efectos secundarios prohibidos

No permitido
  • Cambiar bases de datos
  • Crear/modificar archivos
  • Modificar variables globales
  • Interactuar con interfaces de red
  • Usar console.log
Permitido
  • Leer parámetros de entrada
  • Crear variables locales
  • Llamar otras funciones puras
  • Realizar cálculos matemáticos
  • Retornar nuevos valores

Funciones vs métodos

En Python y POO, se distingue entre funciones y métodos:

Diferencias principales:
  • Pertenencia: función no pertenece a ningún dato; método es propiedad de los datos.
  • Invocación: resultado = funcion(argumento) vs resultado = data.method(arg).
  • Parámetro self: función no lo requiere; método obligado a tener self.
  • Estado: función no cambia entidades; método puede cambiar estado del objeto.
Ejemplo: Math.max(5, 10) es función; array.push(item) es método.

Características clave de la PF

  • Funciones de primer orden y orden superior: tratadas como ciudadanos de primera clase.
  • Inmutabilidad: el valor de una variable no puede cambiar una vez creado.
  • Recursión: en lugar de bucles, se usa recursión para iteraciones.
  • Currificación: transformar función con múltiples argumentos en secuencia de funciones.

Funciones de orden superior

Funciones que aceptan una o más funciones como argumentos y/o devuelven una función.

Analogía: una máquina que puede usar diferentes herramientas según la tarea.
Ejemplo JS: 

const aplicarOperacion = (numeros, operacion) => numeros.map(operacion);

const duplicar = x => x * 2;
console.log(aplicarOperacion([1, 2, 3], duplicar)); // [2, 4, 6]

Inmutabilidad

Si se requiere una modificación, se debe crear una nueva estructura de datos.

Analogía: guardar versiones de un documento sin sobrescribir el original.
Ejemplo JS: 

const numeros = [1, 2, 3];
const nuevos = [...numeros, 4]; // Crea nuevo array
// numeros permanece [1, 2, 3]

Recursión

La recursividad ocurre cuando una función se invoca a sí misma hasta que se cumple una condición de terminación.

Analogía: muñecas rusas donde cada una contiene una más pequeña hasta llegar a la última.
Ejemplo JS: 

const factorial = (n) => {
    if (n <= 1) return 1; // Caso base
    return n * factorial(n - 1); // Llamada recursiva
};

Currificación

Técnica que consiste en transformar una función que usa múltiples argumentos en una secuencia de funciones que usan un único argumento.

Analogía: una máquina de café que primero selecciona el tipo, luego el tamaño, luego los extras.
Ejemplo JS: 

const aplicarImpuesto = (tasa) => (valor) => valor * (1 + tasa);
const aplicarIVA = aplicarImpuesto(0.12);
console.log(aplicarIVA(100)); // 112

Ejercicios prácticos

Para ejemplificar la aproximación funcional, mostraremos la diferencia entre paradigma imperativo y declarativo.

  • Serie Fibonacci con recursividad.
  • Refactorización: suma de números pares.
  • Expresiones lambda con map.
  • Composición de funciones puras.
Objetivo: convertir soluciones imperativas en puramente funcionales usando recursión y funciones de orden superior.

Fibonacci recursivo

Ejemplo clásico donde cada número es la suma de los dos anteriores:

Ejemplo F#: 

let rec fib n = if n < 2 then 1 else fib (n - 1) + fib (n - 2)
Ejemplo JS con memoización: 

const fibonacci = (n, cache = {}) => {
    if (n < 2) return 1;
    if (cache[n]) return cache[n];
    cache[n] = fibonacci(n - 1, cache) + fibonacci(n - 2, cache);
    return cache[n];
};

Refactorización: suma de pares

Imperativo (JS) 

function SumPar(lista) { 
    var resultado = 0; 
    for (var i=0; i< lista.length ; i++) { 
        if (lista[i] % 2 ==0) { 
            resultado += lista[i]; 
        } 
    } 
    return resultado; 
}
Funcional (Haskell) 

sumPar :: Integral a => [a] -> a 
sumPar = (foldl (+) 0) . (filter even)

Composición de filter (solo pares) y foldl (suma con acumulador).

Expresiones lambda

Construcciones de código que generan un valor, especialmente útiles con funciones de orden superior:

Analogía: recetas rápidas que se escriben al momento para una tarea específica.
Ejemplo Python: 

lista1 = [x for x in range(5)]
lista2 = list(map(lambda x: 2 ** x, lista1))
print(lista2) # [1, 2, 4, 8, 16]
Ventaja: hace el código "más corto, más claro y más legible" que funciones tradicionales.
