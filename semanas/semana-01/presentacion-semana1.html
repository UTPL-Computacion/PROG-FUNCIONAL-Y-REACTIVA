Variables y tipos

Recuerda: usa const para valores que no cambian y let cuando planeas reasignar. 

const curso = "Programación Funcional"; // string
let estudiantes = 30;                   // number
let enLinea = true;                     // boolean
let temas = ["Paradigmas", "Funciones"]; // array
let docente = { nombre: "Charlie", grado: "M.Sc." }; // objeto

console.log(curso, estudiantes, enLinea, temas, docente);

Control de flujo básico

Tip: la sintaxis if/else y los bucles tradicionales siguen siendo útiles, aunque pronto los reemplazaremos por expresiones más funcionales. 

const calificacion = 88;

if (calificacion >= 90) {
  console.log("Excelente");
} else if (calificacion >= 70) {
  console.log("Suficiente");
} else {
  console.log("Refuerzo necesario");
}

for (let i = 0; i < 3; i++) {
  console.log(`Iteración ${i}`);
}

Funciones y alcance

Glosario:
  • Función tradicional: se declara con function y define su propio contexto para this.
  • Función flecha (=>): sintaxis corta, no crea un nuevo this ni arguments, ideal para callbacks.
  • ¿Cuándo usar cuál? Usa flechas para funciones internas simples; conserva function cuando necesites tu propio this o hoisting.

function saludar(nombre) {
  return `Hola ${nombre}`;
}

const despedir = (nombre) => `Adiós ${nombre}`;

console.log(saludar("Ana"));
console.log(despedir("Luis"));

Arreglos y métodos útiles

Llave funcional: map/filter/reduce transforman sin mutar el arreglo original.
Glosario rápido:
  • map: crea un nuevo arreglo con el resultado de aplicar una función a cada elemento.
  • filter: devuelve un arreglo con los elementos que cumplen una condición.
  • reduce: combina los elementos en un solo valor acumulado.
  • Mutar: cambiar un valor existente; en programación funcional intentamos evitarlo.

const numeros = [1, 2, 3, 4];

const dobles = numeros.map((n) => n * 2);
const pares = numeros.filter((n) => n % 2 === 0);
const suma = numeros.reduce((acc, n) => acc + n, 0);

console.log("Dobles:", dobles);
console.log("Pares:", pares);
console.log("Suma:", suma);

Objetos y desestructuración

Por qué importa: desestructurar hace más legible el acceso a datos y favorece la inmutabilidad.
Glosario:
  • Inmutabilidad: trabajar con copias en lugar de alterar el original. Evita efectos secundarios.
  • Mutar: modificar un dato existente (por ejemplo, cambiar un elemento de un arreglo).
  • Desestructurar: extraer propiedades de un objeto o elementos de un arreglo en variables nuevas.

const proyecto = {
  nombre: "Calculadora FP",
  versiones: ["alpha", "beta"]
};

const { nombre, versiones } = proyecto;
const versionActual = versiones.at(-1); // última versión

console.log(nombre, versionActual);

Convertir un listado de productos en un catálogo legible:

Tipos de paradigmas

  • Imperativo: Describe paso a paso cómo lograr un resultado controlando el estado del programa.
  • Declarativo: Enfoca el "qué" del resultado, dejando al lenguaje decidir el flujo de ejecución.
  • Funcional: Variante declarativa que se apoya en funciones puras y composición para transformar datos.

Paradigma imperativo

Controla el flujo con instrucciones secuenciales y estados mutables. Ideal cuando necesitas exactitud en cada paso.

Analogía: seguir una receta paso a paso sin saltarse ninguna instrucción.
Ejemplo JS: 

const numeros = [2, 4, 6, 3];
let suma = 0;

for (let i = 0; i < numeros.length; i++) {
  suma += numeros[i];
}

console.log(suma);

Paradigma declarativo

Describe la meta y delega en el lenguaje los pasos intermedios, reduciendo el manejo explícito de estado.

Analogía: indicar el plato que quieres y dejar que el chef decida la preparación.
Ejemplo JS: 

const numeros = [2, 4, 6, 3];
const suma = numeros.reduce((total, numero) => total + numero, 0);

console.log(suma);

Paradigma funcional

Aplica funciones puras y evita mutaciones para transformar datos mediante composición y orden superior.

Analogía: una línea de filtros que transforma un flujo de datos sin alterar el original.
Ejemplo JS: 

const sumarPares = (lista) =>
  lista.filter((numero) => numero % 2 === 0)
       .reduce((total, numero) => total + numero, 0);

console.log(sumarPares([2, 4, 6, 3]));

Programación funcional en 3 ideas

  • Funciones puras: producen el mismo resultado con los mismos datos y no generan efectos secundarios.
  • Inmutabilidad: preferimos crear nuevas versiones de los datos en lugar de modificarlos.
  • Composición: combinamos funciones pequeñas para obtener comportamientos más complejos.

Funciones puras

Mismo input, mismo output, sin efectos secundarios. Facilita pruebas y paralelismo.

Analogía: una máquina expendedora siempre da el mismo producto si presionas el mismo botón.
Ejemplo JS: 

const sumar = (a, b) => a + b;
// sumar(2, 2) siempre devuelve 4 sin tocar nada externo

Inmutabilidad

Los datos no cambian: se crean copias modificadas. Evita estados inconsistentes.

Analogía: guardar versiones de un documento en la nube sin sobrescribir el original.
Ejemplo JS: 

const lenguajes = ["Haskell", "Scala"];
const nuevos = [...lenguajes, "Elixir"];
// lenguajes permanece igual

Composición

Un resultado complejo surge de encadenar funciones pequeñas y reutilizables.

Analogía: cadena de montaje donde cada estación realiza una transformación.
Ejemplo JS: 

const limpiar = (texto) => texto.trim();
const enMayusculas = (texto) => texto.toUpperCase();
const saludo = (nombre) => `Hola ${enMayusculas(limpiar(nombre))}!`;

Profundizando en la teoría

  • Funciones puras: no dependen de variables externas mutables ni generan efectos (como escribir archivos).
  • Inmutabilidad: estructuras de datos persistentes permiten versiones previas sin copias costosas.
  • Composición: `resultado = funcionC(funcionB(funcionA(dato)))` resume un flujo complejo en una sola línea legible.

Imperativo vs funcional

Enfoque imperativo 

let resultado = [];
for (let i = 0; i < numeros.length; i++) {
  if (numeros[i] % 2 === 0) {
    resultado.push(numeros[i] * 10);
  }
}
Enfoque funcional 

const resultado = numeros
  .filter((n) => n % 2 === 0)
  .map((n) => n * 10);

Se lee como una secuencia de transformaciones sobre la colección original.

Ejemplo en JavaScript

Situación: generar un resumen con asistencia al día.


const estudiantes = [
  { nombre: "María", asistencia: true },
  { nombre: "Juan", asistencia: false },
  { nombre: "Ana", asistencia: true }
];

const resumen = estudiantes
  .filter((estudiante) => estudiante.asistencia)
  .map((estudiante) => `✅ ${estudiante.nombre}`);

console.log(resumen); // ["✅ María", "✅ Ana"]

La función original no se modifica y la intención de cada paso es explícita.

Contexto: trabajar con un listado de pedidos e identificar cuáles están listos para despacho.

Código base a refactorizar

Analogía: pasar de una línea de producción manual a una automatizada con estaciones especializadas. 

const pedidos = [
  { id: 101, estado: "listo", total: 45.5 },
  { id: 102, estado: "pendiente", total: 30.0 },
  { id: 103, estado: "listo", total: 27.8 },
  { id: 104, estado: "pendiente", total: 62.1 }
];

let idsListos = [];
let totalListos = 0;

for (let i = 0; i < pedidos.length; i++) {
  const pedido = pedidos[i];
  if (pedido.estado === "listo") {
    idsListos.push(pedido.id);
    totalListos += pedido.total;
  }
}

console.log(idsListos);
console.log(totalListos);
Desafío: reescribir usando composición de funciones. Bonus: crear una función pura que reciba la lista y retorne un objeto con ids y total.

Discusión y extensiones

  • Compartir soluciones: ¿qué funciones de orden superior utilizaron y por qué?
  • Comparar performance: ¿qué tan legible quedó frente a la versión imperativa?
  • Extensión opcional: añadir validaciones (e.g., filtrar pedidos cancelados) sin romper la pureza de las funciones.
Tip: pedir que documenten en comentarios breves la intención de cada transformación.

Mini desafío

Transforma la siguiente lógica a estilo funcional utilizando JavaScript:


lista = [3, 6, 9, 12, 15]
resultado = []
para cada numero en lista:
    si numero > 8:
        agregar numero / 3 a resultado

Pista: usa filter para seleccionar los mayores a 8 y map para dividir cada valor antes de mostrar el resultado.
